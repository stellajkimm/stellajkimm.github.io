Week 2: Inline vs. Inline Block

Inline, Block, and Inline-Block are different ways one can display elements on a page. By default, elements on an html page are displayed in a "block" format.

When elements are displayed in a Block format, each element is thought of as a "block" and naturally stack on top of eachother like so:

html:
<img class="block" src="images/green_square.jpg"/> <img class="block" src="images/blue_square.jpg"/>

css:
.block { display: block; width: 200px;}

When elements are displayed in an Inline format, they sit side-by-side like so:

html:
<img class="block" src="images/green_square.jpg"/> <img class="block" src="images/blue_square.jpg"/>

css:
.block { display: inline; width: 200px;}

This all works out well and makes sense, but what if the element concerned isn't a block element (ie. a paragraph)? Lets find out. Two paragraphs in the block format looks like this:

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Cuius quidem, quoniam Stoicus fuit, sententia condemnata mihi videtur esse inanitas ista verborum. Quid igitur, inquit, eos responsuros putas?

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Cuius quidem, quoniam Stoicus fuit, sententia condemnata mihi videtur esse inanitas ista verborum. Quid igitur, inquit, eos responsuros putas?

html:
<p class="green">Lorem ipsum...</p> <p class="yellow">Lorem ipsum...</p>

css:
.green { background-color: green;} .yellow { background-color: yellow;} p { display: block; width: 200px;}

This is not a surprise. The paragraphs act like blocks and stack on top of eachother. What happens when we make them Inline?

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Cuius quidem, quoniam Stoicus fuit, sententia condemnata mihi videtur esse inanitas ista verborum. Quid igitur, inquit, eos responsuros putas?

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Cuius quidem, quoniam Stoicus fuit, sententia condemnata mihi videtur esse inanitas ista verborum. Quid igitur, inquit, eos responsuros putas?

css:
p { display: inline; width: 200px;}

Woah woah woah. Why aren't they next to eachother in two different rectangles? The reason why is because paragraphs are not block elements so they do not act like blocks. So how do we put the two paragraphs side-by-side then? That is where Inline-Block comes in. Inline-Block is a fusion of Inline and Block when the element concerned isn't exactly shaped as a "block". Inline-Block elements are placed as an inline element but behave like block elements.

This is how it looks when non-block elements are displayed Inline-Block:

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Cuius quidem, quoniam Stoicus fuit, sententia condemnata mihi videtur esse inanitas ista verborum. Quid igitur, inquit, eos responsuros putas?

Lorem ipsum dolor sit amet, consectetur adipiscing elit. Cuius quidem, quoniam Stoicus fuit, sententia condemnata mihi videtur esse inanitas ista verborum. Quid igitur, inquit, eos responsuros putas?

css:
p { display: inline-block; width: 200px;}

So there you have it! I hope this clarifies Inline, Block, and Inline-Block for you!
