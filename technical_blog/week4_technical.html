Week 4: Arrays vs. Hashes

In Ruby, there are two ways to store and organize collections of information: Arrays and Hashes. Arrays are collections of any type of object (integer, string, floating point number, etc) and are ordered and indexed by integers. Hashes are also collections of any type of objects but are indexed by arbitrary keys.

Example of an Array:

array = [1,2,3,4,5]

Example of a Hash:

hash = { "name" => "Stella", "age" => 26, "location" => "Stamford, CT" }

For arrays, you can call on an object by integer-indexing starting with 0. So, that means for our array example, 

array[0]
would be equal to 1, 
array[1]
would be equal to 2, 
array[4]
would be equal to 5 and so forth. Notice, 
array[5]
would be equal to nil because there is no value in the 5th index (there is no 6th object).

For hashes, you can call on the value by calling the keys. So, that means for our hash example, 

hash["name"]
would be equal to "Stella", 
hash["age"]
would be equal to 26, etc.

So how do you know when to use an array or a hash? Well, generally would want to use an array if indexing the objects through ordered integers is fine. However, if you want specific keys of different types to your values, a hash would be more efficient.
