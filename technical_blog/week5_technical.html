Week 5: Classes

When you are writing code for a class, you are basically writing a blueprint from which individual objects can be created. So for example, if you are writing a code for the class "Account", you can create different objects (different accounts) that follow the blueprint. The blueprint can consist of instance variables (things the object knows about itself) and methods (things the object can do). Let's start off with this example:

As you can see, even though this is quite a simple example, there are a lot of things going on here. Let's break it down from the top to bottom.

In the beginning, after defining your class, you must have a initializing method that takes in the instance variables that are important to this class. In this case, it is @name and @balance. Notice, balance is already set equal to 100 by default, but you can always change it by putting in another value for that variable when calling Account.new. The attr_reader is written before the initializing method so that these instance methods can be read anywhere.

After initializing, you can write different methods that the class can do. In this example, this class can call the method display_balance and enter a parameter which is the pin_number that puts the balance in the account if the pin number entered is equal to the pin, or call the method pin_error if it is an incorrect pin. Notice, this instance variable is not listed in attr_reader because it is private and cannot be read elsewhere. Although these methods are private, they can still be used by methods within the class.
